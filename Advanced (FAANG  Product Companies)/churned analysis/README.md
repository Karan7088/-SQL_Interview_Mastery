# 🚀 Churned Users & Revenue Analysis (SQL)

## 🎯 Problem Statement

You are given two tables:

### 1. `subs` (Subscription Table)
Tracks user lifecycle events over time.

| Column       | Description |
|-------------|------------|
| user_id     | Unique user identifier |
| status      | User status (active, paused, resumed, cancelled, inactive) |
| status_time | Timestamp of status change |

---

### 2. `rev` (Revenue Table)
Stores revenue generated by users.

| Column    | Description |
|----------|------------|
| user_id  | Unique user identifier |
| revenue  | Revenue amount |

---

## 🧠 Objective

👉 Identify **churned users** and calculate their **total revenue contribution**

---

## 📌 Churn Definition

A user is considered **churned** if their **latest status** is:

- `cancelled`
- `inactive`

---

## 🧩 Approach

### Step 1: Rank User Status Records

Use `ROW_NUMBER()` to order status events per user:

```sql
ROW_NUMBER() OVER (
    PARTITION BY user_id 
    ORDER BY DATE(status_time)
)
